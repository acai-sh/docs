```md
name: spec-driven-development
description: Follow our conventions for spec-driven development using acai.sh and ACIDs
---

Every feature has a spec file called `FRD.yaml`
FRD files can always be found in `features/<feature-name>`
Each requirement in the spec has an implicit, unique, predictable ID e.g. `SLUG.COMPONENT.1-1`. We call these ACIDs.
The spec is synced to a remote server, where it is broken into individual requirements by ACID, and enriched with metadata (for progress trakcing, coordination, assignment, etc.).

The acai server can be accessed via REST API, CLI or mcp.

```
